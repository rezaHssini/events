# 01 — Vision & Product Overview

## Problem Statement

Event discovery is fragmented. Social platforms show life moments but aren't built for ticketing. Ticketing platforms sell tickets but lack community, content, and delight. Creators juggle Instagram for promotion, Eventbrite for sales, and spreadsheets for seating — with no unified experience for attendees or staff.

## Solution

**Event** is a unified social-events platform where:

- **Creators** run event pages like social profiles — post events with rich media, build followers, sell tickets, configure seating, and manage check-in.
- **Attendees** discover through a personalized feed, interactive map, and powerful filters — then engage socially and attend seamlessly.
- **Staff** scan tickets, manage door lists, and handle on-site operations from dedicated mobile views.
- **The experience** feels modern, animated, and game-like — not corporate.

## Positioning

| Dimension | Position |
|-----------|----------|
| vs Eventbrite | More social, more visual, community-first |
| vs Instagram | Purpose-built for events with native ticketing |
| vs Facebook Events | Modern UX, creator tools, seated venue support |
| vs Partiful/Luma | Fuller platform: pages, commerce, map, staff tools |

## Target Users

1. **Independent creators** — DJs, photographers, workshop hosts, pop-up organizers
2. **Venues & brands** — clubs, galleries, co-working spaces, lifestyle brands
3. **Attendees** — 18–40, mobile-first, values aesthetics and social proof
4. **Event staff** — door staff, volunteers, security, VIP hosts

## Core Value Propositions

### For Attendees
- One app to discover, follow, book, pre-order, and remember events
- See what friends and followed creators are doing
- Browse event menus; pre-order drinks and food; skip the line
- Redeem included perks (welcome drink, tokens) from ticket wallet
- Post photos, videos, and reviews — verified attendee badge
- Map-based exploration with smart filters
- Beautiful ticket experience (wallet passes, animated check-in)

### For Creators
- Event page = social profile + storefront + menu + memory archive
- Rich media storytelling before, during, after events
- Flexible ticketing: free, paid, RSVP, seated, standing
- **F&B menus** with pre-orders and kitchen/bar queue
- **Included perks** per ticket tier (welcome drink, tokens, merch)
- **Social proof flywheel** — past attendee memories sell future events
- Analytics, payouts, and staff management in one place

### For the Platform
- Transaction fees on paid events
- Premium creator subscriptions (advanced analytics, branding)
- Promoted events / discovery ads (Phase 3+)

## Success Metrics (North Stars)

| Metric | Description |
|--------|-------------|
| **GMV** | Gross merchandise value (ticket sales) |
| **Active Creators** | Creators with ≥1 event/month |
| **Attendee MAU** | Monthly active attendees |
| **Follow → Attend Rate** | % of followers who book within 30 days |
| **Check-in Success Rate** | % of tickets successfully scanned |
| **NPS** | Net promoter score (creators + attendees) |

## Key Product Principles

1. **Social first** — Every event is content; every creator is followable
2. **Delight in details** — Micro-animations, haptics, satisfying interactions
3. **Creator empowerment** — Powerful tools without enterprise complexity
4. **Trust & safety** — Verification, moderation, secure payments
5. **Mobile-native** — Primary experience on phone; web for discovery + creator dashboard
6. **Real-world ready** — Offline ticket scanning, geofencing, staff roles

## Monetization Model

| Stream | Model |
|--------|-------|
| Ticket fees | % + fixed fee per paid ticket (passed to attendee or absorbed by creator) |
| Creator Pro | Monthly subscription: advanced analytics, custom branding, priority support |
| Promoted discovery | Sponsored placement in feed/map (clearly labeled) |
| Venue partnerships | Revenue share on venue-claimed pages |

## Competitive Moats

- **Network effects** — Follow graph + social proof ("12 friends going")
- **Creator lock-in** — Event history, follower base, seating templates
- **Data** — Attendee preferences → better recommendations
- **Experience** — Game-like UI differentiation

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Chicken-and-egg (creators vs attendees) | Seed with creator outreach; geo-focused launch |
| Payment/fraud | Stripe Connect, KYC for payouts, velocity limits |
| Content moderation | Report system, AI-assisted review, creator verification |
| Scalability at peak (on-sale moments) | Queue system, seat locking, CDN, read replicas |
| Legal (refunds, taxes) | Configurable refund policies, tax handling per region |

## Launch Strategy (High Level)

1. **City pilot** — Single metro area, 50 creators, nightlife + culture vertical
2. **Creator-led growth** — Invite-only creator onboarding, referral incentives
3. **Attendee viral loops** — Share cards, "friends going", post-event photo albums
4. **Expand vertically** — Workshops, sports, festivals, corporate (later)
